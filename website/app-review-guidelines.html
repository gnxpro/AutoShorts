Application Review Guidelines

This page explains how the GNX Production Desktop application works and how it uses platform APIs. This information is provided for developer platform reviewers.

Application Overview

GNX Production Desktop is a Windows desktop application designed to assist creators and production teams with automated video processing and publishing workflows.

Main Functions

  • Video processing using FFmpeg
  • Importing video sources from local files or URLs
  • Automated content preparation for publishing
  • Media upload to cloud storage providers
  • Publishing or scheduling content to supported platforms

How Platform APIs Are Used

If a platform API is connected, the application may use it for the following purposes:

  • Uploading processed media content
  • Scheduling posts or publications
  • Retrieving account or publishing status information

The application does not access private user data beyond what is required to perform the requested actions.

User Control

Users explicitly connect their accounts and initiate all actions within the desktop application. No automated posting occurs without user configuration.

Data Storage

Most processing occurs locally on the user's device. Only required data is transmitted to third-party services when the user triggers an upload or publishing action.

Security

Authentication tokens and API credentials are stored locally on the user's machine and are not shared with third parties outside of the requested API interaction.
