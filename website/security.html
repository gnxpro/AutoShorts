Security Policy

This document outlines the security practices and architecture of the **GNX Studio** desktop application.

Local-First Architecture

GNX Studio is built as a local-first application. This means the majority of sensitive data processing, video rendering, and credential storage happens directly on your machine, not on our servers.

Data Storage & Encryption

  • Local Storage: Authentication tokens (OAuth) and session data are stored locally in the application's secure directory.
  • No Central Database: We do not maintain a central database of your social media credentials.
  • Configuration: Your project settings and API configurations remain under your physical control.

Secure Transmission

All communication between GNX Studio and third-party APIs (Google, Meta, TikTok) is conducted over encrypted **HTTPS (TLS)** connections. Your data is protected in transit during every upload or sync operation.

Access Control

Integration with external platforms requires explicit user authorization via standard OAuth screens. GNX Studio only performs actions (like uploading or scheduling) that are initiated or configured by the user within the application interface.

Vulnerability Reporting

If you discover a security vulnerability, please contact us immediately at: generalexplorerproduction@gmail.com
