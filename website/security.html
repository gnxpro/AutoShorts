Security Policy

This document explains the security practices used by GNX Production Desktop.

Application Architecture

GNX Production Desktop is primarily a local desktop application. Most processing tasks, including video processing and workflow management, occur directly on the user's device.

Local Data Storage

  • Configuration files are stored locally on the user's machine.
  • Authentication tokens for connected services remain on the user's device.
  • No centralized database is used for storing user credentials.

Data Transmission

When users connect third-party services (such as media platforms or storage providers), communication occurs directly between the user's application and the service using secure HTTPS connections.

Access Control

All integrations with third-party platforms require explicit user authorization. The application only performs actions requested by the user.

Third-Party APIs

GNX Production may integrate with external APIs for publishing workflows, media uploads, or content management. These integrations follow the authentication and security requirements defined by the respective platforms.

Security Updates

The application may periodically check for software updates in order to deliver security fixes and improvements.

Responsible Use

Users are responsible for protecting their local system and ensuring that the application is used in compliance with the policies of connected platforms.

Contact

For security-related questions or vulnerability reports, please contact GNX Production.
